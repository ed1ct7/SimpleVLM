# Лог технических решений (ADR-lite)

Короткие записи: решение + почему. Агенты дописывают сюда, когда делают значимый выбор.

## D1. Рецепт LLaVA (а не end-to-end)
**Решение:** строить VLM по рецепту LLaVA (энкодер + проектор + LLM, 2-стадийное обучение).
**Почему:** проверенный стандарт, дешевле end-to-end (по статье VLMs Explained KOSMOS-2-подход
дороже), deepvk-эталоны построены так же → честное сравнение.

## D2. QLoRA вместо полного файнтюна
**Решение:** обучать через QLoRA (4-bit базовая модель + LoRA-адаптеры).
**Почему:** VRAM всего 16 ГБ. Полный файнтюн 8B не влезает; QLoRA 8B ~11–14 ГБ — помещается
с grad checkpointing и micro-batch.

## D3. Старт с 2B, цель 8B
**Решение:** сначала отладить пайплайн на модели класса 2B (Gemma-2b/Qwen2-2B), потом
целевое обучение Saiga-8b.
**Почему:** быстрый цикл отладки данных/кода без долгих прогонов; 8B — для максимальной
метрики и сравнения с llava-saiga-8b.

## D4. WSL2, не нативный Windows
**Решение:** весь ML-стек в WSL2 (Ubuntu).
**Почему:** bitsandbytes/flash-attn/deepspeed на нативном Windows нестабильны; в WSL2 всё
ставится как на Linux, RTX 5080 виден через CUDA.

## D5. Свежий стек под Blackwell
**Решение:** CUDA 12.8+, PyTorch 2.7+ (cu128), свежие bitsandbytes/triton.
**Почему:** RTX 5080 — Blackwell (sm_120). Старые сборки падают `no kernel image available`.
