## Где ты

Ты — агент-исполнитель в **мульти-сессийном** проекте «Обучение визуально-языковой модели
(VLM) для VK». Сессии стартуют холодными и передают работу через файлы на диске, не через
память чата.

## Первые два шага (всегда)

1. Прочитай **`CONTEXT.md`** — задача, открытые данные VK, архитектура, стек, железо,
   footgun'ы (Blackwell/WSL2/VRAM 16 ГБ).
2. Прочитай **`STATE.md`** — что уже сделано, какие артефакты на диске, какая задача твоя.

## Правила хэндоффа

- Твой бриф — в `prompts/NN-*.md`. Делай строго по нему (Цель → Вход → Что сделать → Выход
  → Критерий приёмки).
- Артефакты пиши в **`solution/`** (своя подпапка), не «в чат».
- В конце **допиши `STATE.md`** (append-only): статус задачи + что разблокировал. Историю не
  переписывай.
- Значимый технический выбор фиксируй в `docs/decisions.md`.
- Факты не копируй в свои файлы — ссылайся на `CONTEXT.md`.
- Противоречие промпта и `CONTEXT.md` → не угадывай, сообщи оркестратору, пометь ⛔ в STATE.

## Жёсткие ограничения железа (полное — CONTEXT §4)

- GPU RTX 5080, **16 ГБ VRAM** — стена. RAM (64 ГБ)/CPU её не двигают.
- 8B только через **QLoRA + grad checkpointing + micro-batch**.
- Стек ставить в **WSL2**, свежий (CUDA 12.8+, PyTorch 2.7+, Blackwell sm_120).
- Данные — в WSL-ФС (`~/...`), не на `/mnt/c|d`.
