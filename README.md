# VK-VLM — оркестрация проекта «Обучение визуально-языковой модели»

Этот репозиторий — **управляющий слой** для выполнения проекта VK силами нескольких
агент-сессий. Сам код решения живёт в `solution/`, а здесь — контекст, план, промпты и
трекер состояния, по которым холодные сессии подхватывают работу.

## Как пользоваться (для оркестратора)

1. Открой `STATE.md` — увидишь, что сделано и какая задача следующая.
2. Возьми соответствующий файл из `prompts/NN-*.md`.
3. Отдай его новой агент-сессии **целиком, как есть** (он самодостаточен).
4. Агент читает `CONTEXT.md` + `STATE.md`, выполняет, пишет артефакты в `solution/`,
   дописывает `STATE.md`.
5. Повторяй по порядку зависимостей (см. `prompts/00-orchestrator.md`).

## Как пользоваться (для агента, который получил промпт)

1. **Сначала прочитай `CONTEXT.md`** — это источник правды (задача, данные, стек, решения).
2. Прочитай `STATE.md` — что уже готово и какие артефакты есть на диске.
3. Выполни задачу из своего промпта. Артефакты складывай в `solution/`.
4. В конце **допиши `STATE.md`**: статус задачи + что разблокировал для следующих.

## Структура

```
.                       ← корень репозитория (рабочая папка сессии)
├── CLAUDE.md           ← boot-loader агент-сессий (авто-грузится первым)
├── README.md           ← этот файл
├── CONTEXT.md          ← ИСТОЧНИК ПРАВДЫ (читать первым)
├── PLAN.md             ← полный 10-этапный план по алгоритму VK
├── STATE.md            ← живой трекер прогресса (append-only)
├── prompts/            ← самодостаточные брифы по задачам
│   ├── 00-orchestrator.md
│   ├── 01-env-setup.md
│   ├── 02-data-pipeline.md
│   ├── 03-train-2b.md
│   ├── 04-train-8b.md
│   ├── 05-eval.md
│   └── 06-report.md
├── solution/           ← код и артефакты решения (env, data, train, model)
└── docs/
    ├── datasets.md     ← разбор открытых данных deepvk
    └── decisions.md    ← лог технических решений
```

## Главные правила хэндоффа

- **Каждый промпт самодостаточен** — сессии стартуют без памяти прошлых.
- **Факты не дублировать** — единственный источник правды `CONTEXT.md`, остальное ссылается.
- **`STATE.md` — append-only** — не переписывать историю, только дописывать.
- **Артефакты на диск в `solution/`**, не «в чат» — следующий агент берёт с диска.
- **Критерий приёмки** в каждом промпте — агент сам понимает, когда «готово».
