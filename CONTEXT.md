# CONTEXT — источник правды проекта VK-VLM

> Спецификация проекта — единый источник правды. Здесь факты, которые не меняются от
> задачи к задаче (задача, данные, архитектура, стек, железо, цель).

## 1. Задача

Обучить русскоязычную визуально-языковую модель (VLM) на **открытых данных VK** и
получить **максимальную метрику** на двух бенчмарках: `GQA-ru` и `MMBench-ru`.
Сравнить с эталонными моделями deepvk.

VLM = на вход изображение + текст, на выход текст. Архитектура из 3 частей:
**визуальный энкодер → проектор (MLP) → языковая модель (LLM)**. Рецепт обучения — LLaVA.

## 2. Открытые данные (коллекция deepvk на Hugging Face)

| Ресурс | Тип | Размер | Роль |
|---|---|---|---|
| `deepvk/LLaVA-Instruct-ru` | датасет | 144k | **обучение** (SFT) |
| `deepvk/GQA-ru` | бенчмарк | 80.1k | **оценка**, открытые вопросы, accuracy |
| `deepvk/MMBench-ru` | бенчмарк | 3.91k | **оценка**, выбор A/B/C/D (20 навыков), accuracy |
| `deepvk/llava-saiga-8b` | модель | 8B | **эталон** (Saiga-8b + LLaVA) |
| `deepvk/llava-gemma-2b-lora` | модель | 3B | **эталон** (Gemma-2b + LoRA) |

### Критичный нюанс про изображения
`LLaVA-Instruct-ru` поля: `type`, `conversations`, `id` (id картинки в COCO),
`image` (путь к картинке в COCO). **Файлов изображений внутри НЕТ** — только привязка.
Картинки берутся из открытого набора **COCO 2014** (train/val split совпадает с датасетом),
качаются отдельно с официального источника. train2014 ~13 ГБ, val2014 ~6 ГБ.

## 3. Архитектура и метод

- **Энкодер:** CLIP ViT-L/14 или **SigLIP** (SigLIP часто даёт прирост).
- **Проектор:** 2-слойный MLP.
- **LLM:** старт `Gemma-2b` / `Qwen2-2B` (быстрый цикл), цель `Saiga-8b` (максимум).
- **Рецепт LLaVA, 2 стадии:**
  1. энкодер + LLM заморожены, обучается только проектор;
  2. совместное дообучение проектора + LLM (через QLoRA).
- **Метод экономии памяти:** QLoRA (4-bit) + gradient checkpointing + micro-batch +
  gradient accumulation.

## 4. Оборудование и окружение

- **GPU:** NVIDIA GeForce RTX 5080, **16 ГБ VRAM**, архитектура **Blackwell (sm_120)**.
- **RAM:** 64 ГБ. **CPU:** 16 ядер / 32 потока.
- **ОС:** Windows 11 + **WSL2 (Ubuntu)** — весь ML-стек ставить в WSL2.
- **Что влезает:** модель 2B свободно; **8B только через QLoRA** (~11–14 ГБ) с grad
  checkpointing и micro-batch.

### Footgun'ы железа (НЕ игнорировать)
- **Blackwell = новый стек.** Старые сборки падают с `no kernel image is available`.
  Нужно: **CUDA 12.8+**, **PyTorch 2.7+** (cu128), свежие `bitsandbytes` (Blackwell-ядра),
  `triton`, при необходимости `flash-attn`/`xformers` под sm_120.
- **Ставить в WSL2, не нативный Windows.** Нативный Windows + bitsandbytes/flash-attn = боль.
- Данные держать в WSL-ФС (`~/...`), **не** на `/mnt/c|d` (медленный I/O).
- AMD Radeon (встройка) для ML не использовать — только NVIDIA.

## 5. Стек ПО

`transformers`, `TRL` (`SFTTrainer` — поддерживает VLM), `peft` (LoRA), `bitsandbytes`
(4-bit), `accelerate`, `datasets`. Оценка — `lmms-eval` или собственный скрипт.

## 6. Цель и критерий результата

**Цель проекта:** обучить рабочую русскоязычную VLM на открытых данных VK по рецепту LLaVA
(энкодер → проектор → LLM), воспроизвести архитектуру эталона deepvk и **честно измерить**
её на обоих бенчмарках (GQA-ru, MMBench-ru) единым воспроизводимым протоколом; разрыв с
эталоном — проанализировать и наметить путь сокращения. (VK ставит обязательными 4 требования
сдачи — §8; «максимальная метрика» в брифе — пример цели, не порог. Цель ставим сами.)

**Критерий результата (что считаем выполненным):** на диске есть обученная модель, связно
отвечающая по-русски по картинке; eval даёт числа на обоих бенчмарках для своей модели и
эталонов в одном протоколе; разрыв объяснён.

**Stretch-цель (опционально, не для базовой сдачи):** догнать/обойти эталон deepvk своего
класса. Главный рычаг — обучение на полном манифесте (подвыборка → весь корпус, ±эпохи).
