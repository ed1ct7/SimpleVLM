# Задача 04: Целевое обучение модели 8B (Saiga-8b, QLoRA, 2 стадии)

## Контекст
Прочитай `CONTEXT.md` (§3, §4, §6) и `docs/decisions.md`. Проверь `STATE.md`: задача 03 ✅,
пайплайн обучения отлажен и параметризован.

## Цель
Обучить целевую русскоязычную VLM на базе `Saiga-8b` по полному 2-стадийному рецепту LLaVA
с QLoRA, на полном (или большом) объёме `LLaVA-Instruct-ru`.

## Вход
- Отлаженный пайплайн из 03 (`solution/train/`), данные из 02.
- Базовая LLM: `Saiga-8b` (или лучшая по результатам 03). Энкодер: выбранный в 03.

## Что сделать
1. Стадия 1 (alignment): энкодер и LLM **заморожены**, обучается **только проектор**.
   Цель — выровнять визуальные признаки с пространством LLM.
2. Стадия 2 (instruction tuning): совместное дообучение проектора + LLM через **QLoRA**
   на инструкционных данных `LLaVA-Instruct-ru`.
3. Память под 16 ГБ VRAM: 4-bit, `gradient_checkpointing`, `batch=1` +
   `gradient_accumulation` (подобрать под лимит), картинки 336px. При нехватке — CPU-offload
   оптимизатора (есть 64 ГБ RAM).
4. Логировать loss, сохранять чекпойнты, фиксировать гиперпараметры в конфиг.
5. Сохранить финальные адаптеры + проектор; проверить инференс на нескольких примерах.

## Выход (артефакты)
- `solution/train/configs/8b-stage1.yaml`, `8b-stage2.yaml` — конфиги стадий.
- `solution/model/8b/` — финальные LoRA-адаптеры + проектор (или ссылка на HF Hub, если
  большие).
- `solution/train/RUN-8b.md` — лог запуска: команды, время, пиковая VRAM, гиперпараметры.

## Критерий приёмки
- [ ] Обе стадии прошли без OOM/sm_120-ошибок, чекпойнты сохранены.
- [ ] Loss снижается; финальная модель грузится и отвечает связно по-русски.
- [ ] Гиперпараметры и параметры запуска задокументированы (воспроизводимость).

## Подводные камни
- 8B в 16 ГБ — впритык: начать с короткого прогона на подвыборке, убедиться в стабильности
  памяти, потом полный объём.
- Долгое обучение → запускать в фоне/`tmux`, логировать в файл.
