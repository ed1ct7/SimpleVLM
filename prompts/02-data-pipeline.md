# Задача 02: Пайплайн данных (LLaVA-Instruct-ru + COCO 2014)

## Контекст
Прочитай `CONTEXT.md` (особенно §2 — нюанс с изображениями) и `docs/datasets.md`.
Проверь `STATE.md`, что задача 01 (окружение) ✅.

## Цель
Получить готовый к обучению набор: тексты `LLaVA-Instruct-ru` + сопоставленные с ними
изображения COCO 2014, плюс лоадер, отдающий пары (изображение, диалог).

## Вход
- Окружение из задачи 01 (WSL2, активированное venv/conda).
- Датасет `deepvk/LLaVA-Instruct-ru` (HF). Поля: `type`, `conversations`, `id`, `image`.
- Изображения отдельно: COCO 2014 (`train2014` ~13 ГБ, `val2014` ~6 ГБ) с cocodataset.org.

## Что сделать
1. Скрипт `solution/data/download.py`: тянет `deepvk/LLaVA-Instruct-ru` через `datasets`;
   качает и распаковывает COCO `train2014`/`val2014` в `~/vk-vlm-data/coco/` (НЕ на /mnt).
2. Скрипт `solution/data/build_dataset.py`: по полю `image` сопоставляет запись датасета с
   файлом картинки; отбрасывает записи без файла (с логом, сколько отброшено).
3. Привести `conversations` к chat-формату целевой LLM (роль user/assistant, плейсхолдер
   изображения). Оставить выбор LLM-шаблона параметром.
4. Скрипт-проверка `solution/data/sample.py`: берёт 5 случайных примеров, печатает текст +
   подтверждает, что картинка открывается (PIL), размеры > 0.
5. Добавить `solution/.gitignore`: исключить `~/vk-vlm-data`, COCO, чекпойнты, `.venv`,
   `__pycache__` (данные не коммитить).

## Выход (артефакты)
- `solution/data/download.py`, `build_dataset.py`, `sample.py`.
- `solution/data/README.md` — как запустить загрузку, куда ложатся данные, статистика
  (сколько записей, сколько с валидной картинкой).
- `solution/.gitignore`.
