# Разбор открытых данных (этап 5)

Источник — коллекция **deepvk** «Vision-Language Modeling» на Hugging Face. Все данные
открытые. Ниже — что есть, схема, роль в проекте.

## Обучающий датасет

### `deepvk/LLaVA-Instruct-ru` — 144k записей
GPT-сгенерированный инструкционный датасет в стиле LLaVA, на русском. **Топливо тренировки
(SFT).**

Поля каждой записи:
| Поле | Что |
|---|---|
| `type` | тип примера: `conversation` или `complex_reasoning` |
| `conversations` | список реплик: словари с ключами `from` и `value` |
| `id` | id картинки в COCO (не уникален) |
| `image` | путь к картинке в COCO |

**Изображений внутри нет** — только привязка к COCO. См. ниже.

### Изображения — COCO 2014 (внешний открытый набор)
- Источник: cocodataset.org. Нужны `train2014` (~13 ГБ) и `val2014` (~6 ГБ).
- train/val split в `LLaVA-Instruct-ru` соответствует split COCO 2014.
- Сопоставление: по полю `image` (напр. `COCO_train2014_000000123456.jpg`) находишь файл.
- Скачивается отдельно скриптом на этапе `02-data-pipeline`.

## Бенчмарки (оценка)

### `deepvk/GQA-ru` — 80.1k
Перевод GQA. Открытые вопросы по сцене (отношения объектов, атрибуты, логика). Метрика —
**accuracy** (exact-match ответа). Большой → для оценки берут стандартный сабсет.

### `deepvk/MMBench-ru` — 3.91k
Перевод MMBench. Вопросы с **выбором (A/B/C/D)**, 20 навыков: OCR, локализация объектов,
распознавание, рассуждение. Метрика — **accuracy** по выбору. Чувствителен к формату
промпта — модель должна выдать букву; оригинал использует circular eval.

## Эталонные модели (сравнение, не обучение)

### `deepvk/llava-saiga-8b` — 8B
VLM на базе Saiga-8b, дообучена в сетапе LLaVA. Главный baseline по качеству.

### `deepvk/llava-gemma-2b-lora` — 3B
VLM на базе Gemma-2b, дообучена с LoRA в сетапе LLaVA. Лёгкий baseline.

**Использование:** прогнать обе на `GQA-ru` и `MMBench-ru`, получить целевые числа, к
которым стремиться своей моделью.
